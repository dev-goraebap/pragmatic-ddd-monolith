# harden

> 트랜잭션·동시성·에러·멱등성·재시도 등 운영 견고성을 보강한다. 카테고리: Refine.

## 사용 시점

- 베타 → 프로덕션 승격 직전.
- 장애 회고 후 동일 패턴 재발 방지.
- 외부 결제·메시징·웹훅 같이 실패 가능성이 높은 경계.

## 입력

- 견고화 대상 유스케이스.
- 알려진 실패 모드(이중 클릭, 네트워크 단절, DB 데드락 등).

## 절차

1. (TBD) **트랜잭션 경계 재확인**: 한 UoW에 적정 범위가 묶였는가. 외부 호출이 트랜잭션 내부에 있는가.
2. (TBD) **외부 호출 격리**: 트랜잭션 내부에서 외부 API 호출 금지. Outbox 또는 사후 트리거로.
3. (TBD) **멱등성 키**: 동일 요청 재시도 안전한가. 키 컬럼·헤더 도입.
4. (TBD) **동시성 충돌**: 낙관적 락(`version`) 또는 비관적 락 선택.
5. (TBD) **에러 분류 보강**: `Conflict` / `Failed Precondition` 자리 메우기.
6. (TBD) **재시도 정책**: 어떤 에러에서 재시도하고 어디서 멈출지.
7. (TBD) **타임아웃**: 외부 호출별 명시.
8. (TBD) 운영 시나리오 테스트(통합 또는 카오스).

## 출력

- 견고화된 유스케이스와 회귀 안전망.

## 관련

## 프레임워크 노트

- NestJS: ALS 기반 UoW가 트랜잭션 누수 막는 핵심. Outbox는 별도 모듈.
- Spring: `@Transactional` propagation 옵션 명시, `@Retryable` 활용.
- ASP.NET Core: `TransactionScope` + Polly로 재시도/회로 차단.
